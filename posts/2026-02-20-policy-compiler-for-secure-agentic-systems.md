# 에이전트 보안 정책 컴파일러

**부제:** Policy Compiler for Secure Agentic Systems

## 한 줄 결론
PCAS는 결정적 정책 집행을 제공하여 LLM 기반 에이전트 시스템의 보안을 크게 강화한다.

## TL;DR (요약)
- LLM(대규모 언어 모델) 기반 에이전트가 복잡한 승인 및 데이터 접근 정책을 준수하도록 보장하기 어려움.  
- PCAS는 에이전트 시스템 상태를 이벤트 간 인과 관계를 표현하는 의존성 그래프(dependency graph)로 모델링.  
- Datalog에서 파생된 선언형 정책 언어로 정보 흐름과 교차 에이전트 출처(provenance)를 기술.  
- 참조 모니터(reference monitor)가 모든 액션을 가로채 위반을 차단해 결정적(policy-by-construction) 집행 보장.  
- 프롬프트 삽입 공격 방어, 다중 에이전트 승인 워크플로우, 고객 서비스 정책 등 실험에서 정책 준수율을 48%에서 93%로 향상.

## 문제 정의(Problem)
LLM 기반 에이전트 시스템이 고객 응대 프로토콜, 승인 워크플로우, 데이터 접근 제한, 규제 준수 등 복잡한 권한 정책을 필요로 하지만, 현재는 단순히 프롬프트에 정책을 삽입하는 방식으로 처리하고 있어 실행 시 집행 보장이 없다. 메시지 내역만을 선형으로 추적하는 접근법으로는 에이전트 간 정보 흐름과 도구 호출(tool call)·결과·메시지 이벤트 간 인과 관계를 포착하기 어렵다.

## 제안 방법(Method)
PCAS(Policy Compiler for Agentic Systems)는 다음과 같은 단계로 정책 집행을 구현한다.  
- 에이전트 시스템의 상태를 이벤트 간 인과 관계를 표현하는 의존성 그래프(dependency graph)로 모델링.  
- Datalog에서 파생된 정책 언어를 사용해 트랜지티브(전이적) 정보 흐름과 교차 에이전트 출처(provenance)를 선언형 규칙(rule)으로 명시.  
- 참조 모니터(reference monitor)가 모든 도구 호출, 결과 수신, 메시지 전송 등의 액션을 가로채 정책 위반을 사전에 탐지·차단.  
- 기존 에이전트 구현체와 정책 사양(specification)을 입력하면, 별도의 보안 특화 구조 변경 없이도 정책 준수 시스템으로 자동 변환(compile)하여 출력.

## 핵심 기여/차별점(Contributions)
- 인과 관계 기반 의존성 그래프 모델링: 도구 호출, 결과, 메시지 이벤트 간 트랜스액션 정보를 명확히 추적  
- Datalog 파생 선언형 정책 언어: 정보 흐름과 에이전트 간 출처를 포괄적으로 기술하여 복잡한 권한 규칙도 표현 가능  
- 참조 모니터 기반 결정적 집행: 모델 추론에 의존하지 않는 강제적(policy-by-construction) 정책 위반 차단 메커니즘

## 한계/리스크(Limitations)
초록 기준으로는 PCAS가 대규모 배포 환경에서 보이는 성능 오버헤드, 정책 언어의 표현 한계, 그래프 유지·관리 비용 등 구체적 한계 및 리스크를 확인할 수 없다.

## 실무 적용 아이디어(Practical Takeaways)
- 복잡한 승인 워크플로우나 데이터 접근 제어가 필요한 멀티에이전트 시스템에 의존성 그래프 기반 추적 도입  
- 에이전트 실행 전후에 참조 모니터를 배치하여 모든 API·도구 호출을 가로채 정책 준수 여부 검사  
- 정책 요구사항을 Datalog 유사 선언형 언어로 정리하여 개발자와 보안 담당자가 공통으로 검토·관리
